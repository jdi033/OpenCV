# YOLOv8 自定义模型分类分支训练失败根因分析

## 1. 问题现象

使用 COCO128 数据集（128 张图片、80 个类别）训练自定义 YOLOv8 模型 300 个 epoch 后，推理时图片中检测不到任何目标。

诊断数据：
- 分类 logits 全部为 **0.0000**（应有正有负）
- sigmoid 后全部为 **0.5000**（无区分度）
- 回归分支 LTRB 值范围 0~15（回归分支正常学习）
- 分类分支最后一层权重全部为负值（min=-0.46, max=-0.09）
- 分类分支中间激活值范围 -13.78 ~ -5.64（强烈负偏）

---

## 2. 涉及的核心模块

### 2.1 Detect 检测头

Detect 模块包含两个分支：

```
分类分支 (cv2): Conv → BN → SiLU → Conv → BN → SiLU → Conv2d(nc, 1)
回归分支 (cv3): Conv → BN → SiLU → Conv → BN → SiLU → Conv2d(4*reg_max, 1)
```

训练时返回原始 logits（不经过 sigmoid）：
```python
# 训练模式
return cls_concatenated.permute(0, 2, 1), reg_concatenated.permute(0, 2, 1)
# cls: [B, 8400, 80]  原始 logits
# reg: [B, 8400, 64]  原始分布特征
```

推理时对分类分支施加 sigmoid：
```python
# 推理模式
cls_scores = cls_concatenated.sigmoid()  # logits → [0, 1]
reg_results = self.dfl(reg_concatenated)  # 64维分布 → 4维距离
out = torch.cat((cls_scores, reg_results), 1)  # [B, 84, 8400]
```

### 2.2 TaskAlignedAssigner 任务对齐分配器

负责将 GT 框分配给最匹配的预测点。核心公式：

```python
alignment_metrics = (scores ** alpha) * (ious ** beta) * mask_gt
```

参数含义：
- `scores`：预测点对 GT 类别的分类置信度（sigmoid 后），形状 [B, M, N]
- `ious`：预测框与 GT 框的交并比，形状 [B, M, N]
- `mask_gt`：有效 GT 框的掩码（排除 padding 填充框），形状 [B, M, N]
- `alpha = 0.5`：分类得分的指数权重
- `beta = 6.0`：IoU 的指数权重
- `topk = 10`：每个 GT 框选取的候选正样本数

分配流程：
1. 计算每个预测点与每个 GT 框的对齐分数
2. 对每个 GT 框，选取 topk 个最高对齐分数的预测点作为正样本候选
3. 如果一个预测点同时被多个 GT 框选中，只保留对齐分数最高的那个
4. 最终输出 `mask_pos`（正样本掩码）和 `alignment_metrics`（对齐分数）

### 2.3 v8DetectionLoss 损失函数

分类损失计算：
```python
# target_scores: [B, 8400, 80]，初始全 0
# 对正样本位置赋值为 alignment_metrics 的最大值
target_scores[b, pos_idxs, pos_target_labels] = max_metrics[b, pos_idxs]

# 分类损失 = BCE(pred_logits, target_scores) / 正样本总数
loss_cls = BCEWithLogitsLoss(pred_scores, target_scores).sum() / target_scores_sum
```

回归损失计算：
```python
# 对每个正样本，计算预测框与对应 GT 框的 IoU
loss_box = (1 - IoU).sum() / 正样本总数
```

总损失：
```python
return loss_cls + loss_box * 1.5
```

## 3. 失败机制的数学推导

### 3.1 对齐分数随分类置信度的变化

beta=6.0 使 IoU 成为对齐分数的主导因素。以下表格展示不同 scores 和 ious 组合下的 alignment_metrics：

| scores (sigmoid后) | ious | scores^0.5 | ious^6.0 | alignment_metrics |
|:---:|:---:|:---:|:---:|:---:|
| 0.500 | 0.3 | 0.707 | 0.000729 | 0.000516 |
| 0.500 | 0.5 | 0.707 | 0.015625 | 0.011049 |
| 0.500 | 0.7 | 0.707 | 0.117649 | 0.083218 |
| 0.100 | 0.3 | 0.316 | 0.000729 | 0.000231 |
| 0.100 | 0.5 | 0.316 | 0.015625 | 0.004940 |
| 0.100 | 0.7 | 0.316 | 0.117649 | 0.037200 |
| 0.010 | 0.3 | 0.100 | 0.000729 | 0.000073 |
| 0.010 | 0.5 | 0.100 | 0.015625 | 0.001563 |
| 0.010 | 0.7 | 0.100 | 0.117649 | 0.011765 |
| 0.001 | 0.3 | 0.032 | 0.000729 | 0.000023 |
| 0.001 | 0.5 | 0.032 | 0.015625 | 0.000494 |
| 0.001 | 0.7 | 0.032 | 0.117649 | 0.003718 |

观察：当 scores 从 0.5 降到 0.001 时，即使 ious=0.7，alignment_metrics 也从 0.083 降到 0.004。target_scores 随之下降约 22 倍。

### 3.2 分类损失随 logits 的变化

BCEWithLogitsLoss 的数学定义：

```
L = -t × log(σ(z)) - (1-t) × log(1-σ(z))
```

其中 `z` 是预测 logits，`t` 是目标值，`σ` 是 sigmoid 函数。

当 target_scores 趋近 0 时（t ≈ 0）：

```
L ≈ -(1-t) × log(1-σ(z))
  ≈ -1 × log(1-σ(z))
  = -log(σ(-z))
```

以下表格展示不同 logits 值对应的单点分类损失（假设 target ≈ 0）：

| logits z | sigmoid(z) | sigmoid(-z) | 单点损失 -log(σ(-z)) |
|:---:|:---:|:---:|:---:|
| 0.0 | 0.5000 | 0.5000 | 0.6931 |
| -1.0 | 0.2689 | 0.7311 | 0.3133 |
| -2.0 | 0.1192 | 0.8808 | 0.1269 |
| -3.0 | 0.0474 | 0.9526 | 0.0485 |
| -5.0 | 0.0067 | 0.9933 | 0.0067 |
| -7.0 | 0.0009 | 0.9991 | 0.0009 |
| -9.0 | 0.0001 | 0.9999 | 0.0001 |
| -9.27 | 0.0001 | 0.9999 | 0.0001 |

观察：当 logits 从 0 降到 -9.27 时，单点损失从 0.693 降到 0.0001，下降约 **6900 倍**。

### 3.3 梯度消失的数学证明

BCEWithLogitsLoss 对 logits z 的梯度：

```
∂L/∂z = σ(z) - t
```

当 target t ≈ 0 时：

```
∂L/∂z ≈ σ(z)
```

| logits z | sigmoid(z) = 梯度 |
|:---:|:---:|
| 0.0 | 0.5000 |
| -2.0 | 0.1192 |
| -5.0 | 0.0067 |
| -7.0 | 0.0009 |
| -9.0 | 0.0001 |
| -9.27 | 0.0001 |

当 logits = -9.27 时，梯度仅为 0.0001。这意味着每个预测点对分类分支权重的更新量极小。

在一次前向传播中，分类分支的总梯度是所有预测点梯度的累积。但由于负样本（背景点）和正样本（目标点）的梯度方向相反：

- 负样本（约 107 万个）：推动 logits 向负方向移动（正确方向）
- 正样本（约几千个）：推动 logits 向正方向移动（被需要的方向）

当正样本的 target_scores 也很低时（因为 alignment_metrics 包含 scores 项），正样本的梯度信号被严重削弱，无法对抗负样本的"拉力"。

## 4. 自我强化的死循环

### 4.1 循环的五个阶段

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐          │
│   │ 分类 logits │ ──→ │ sigmoid  │ ──→ │  scores  │          │
│   │  负值增大   │     │  趋近 0  │     │  趋近 0  │          │
│   └──────────┘     └──────────┘     └────┬─────┘          │
│         ↑                                │                 │
│         │                                ↓                 │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐          │
│   │  梯度消失  │ ←── │  损失降低  │ ←── │ target   │          │
│   │ 权重不更新  │     │  趋近 0  │     │ scores低 │          │
│   └──────────┘     └──────────┘     └──────────┘          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**阶段 1 — 初始状态（Epoch 0）**

模型权重随机初始化，分类 logits ≈ 0，sigmoid ≈ 0.5。
此时 scores=0.5，alignment_metrics 不算太低，分配器能正常工作。
正样本的 target_scores ≈ 0.05~0.08，分类损失有梯度。

**阶段 2 — 负样本主导（Epoch 1~10）**

128 张图 × 8400 预测点 = 约 107 万个预测点。
其中正样本仅几千个，负样本超过 106 万个。
负样本的 BCE 损失推动所有 logits 向负方向移动。

负样本梯度的数学推导：
```
对于负样本（target=0）：∂L/∂z = σ(z) - 0 = σ(z)
当 z=0 时，每个负样本的梯度 = 0.5
总负梯度 ≈ 1,070,000 × 0.5 = 535,000
```

正样本梯度的数学推导：
```
对于正样本（target=t）：∂L/∂z = σ(z) - t
当 z=0, t=0.05 时，每个正样本的梯度 = 0.5 - 0.05 = 0.45
假设正样本 3000 个：总正梯度 ≈ 3000 × 0.45 = 1,350
```

正负梯度比 = 1,350 / 535,000 ≈ 0.0025。正样本的梯度信号仅为负样本的 0.25%。
在梯度累积中，负样本完全主导，logits 快速向负方向移动。

**阶段 3 — 进入死循环（Epoch 10~50）**

logits 下降到 -3 ~ -5 区间，sigmoid ≈ 0.007~0.05。
alignment_metrics 中的 scores 项使 target_scores 也下降。
损失开始减小，梯度进一步缩小。

此时正样本梯度：
```
当 z=-5, t=0.007 时：∂L/∂z = σ(-5) - 0.007 = 0.0067 - 0.007 ≈ -0.0003
每个正样本梯度仅 0.0003，3000 个正样本总梯度 ≈ 0.9
```

负样本梯度：
```
当 z=-5 时：∂L/∂z = σ(-5) = 0.0067
106 万个负样本总梯度 ≈ 7,102
```

正负梯度比进一步恶化到 0.00013。分类分支几乎只受负样本影响。

**阶段 4 — 梯度消失（Epoch 50~200）**

logits 下降到 -7 ~ -9 区间，sigmoid ≈ 0.0001~0.001。
损失趋近 0，梯度趋近 0。

此时：
```
每个预测点的梯度 = σ(-9) ≈ 0.0001
107 万个预测点的总梯度 ≈ 107
```

相比阶段 2 的总梯度 535,000，下降了约 5000 倍。
权重更新量 = 梯度 × 学习率 = 107 × 0.001 = 0.107
对于一个拥有数万参数的分类分支，每次更新量 0.107 几乎可以忽略。

**阶段 5 — 完全停滞（Epoch 200~300）**

logits 稳定在 -9.27 附近，sigmoid ≈ 0.0001。
损失 ≈ 0.0001，梯度 ≈ 0.0001。
模型停止学习，权重几乎不再变化。

验证数据：
- Epoch 50 与 Epoch 300 的分类权重最大差异仅 0.087
- 分类分支中间激活值范围 -13.78 ~ -5.64
- 分类分支最终输出全部为 0

### 4.2 为什么回归分支没有陷入同样的困境

回归分支不依赖 sigmoid 激活，其损失基于 DFL（分布焦点损失）+ IoU：

```python
# DFL：将 64 维分布转换为 4 维距离
pred_ltrb = DFL(pred_dist)

# 距离 × 步长 = 像素距离
pred_ltrb = pred_ltrb * stride_tensor

# 转换为绝对坐标
pred_bboxes = dist2bbox(pred_ltrb, anchor_points)

# IoU 损失
loss_box = (1 - IoU).sum() / 正样本数
```

回归损失的梯度不经过 sigmoid，不受"低置信度死循环"影响。
即使分类得分很低，只要预测框与 GT 框有几何重叠，IoU 损失就能提供有效的学习信号。

因此诊断数据中：
- 回归分支 LTRB 值范围 0~15（正常学习）
- 分类分支 logits 全部为 0（完全停滞）

## 5. COCO128 数据集的结构性缺陷

### 5.1 数据统计

| 指标 | 数值 |
|------|------|
| 训练图片数 | 128 |
| 总标注框数 | 941 |
| 有标注的类别数 | 71 / 80 |
| 完全缺失的类别 | 10, 12, 18, 19, 37, 47, 66, 70, 78（共 9 个） |
| 每张图平均标注框数 | 7.4 |
| 每个类别平均标注框数 | 13.3 |
| 每个类别平均出现的图片数 | ~10-15 张 |

### 5.2 正样本数量估算

YOLOv8 的检测头在 640×640 输入下产生 8400 个预测点（80×80 + 40×40 + 20×20）。
TaskAlignedAssigner 为每个 GT 框分配 topk=10 个正样本。

```
每张图正样本数 ≈ 7.4 个 GT 框 × 10 = 74 个
每个 batch 正样本数 ≈ 74 × 4（batch_size）= 296 个
每个 batch 负样本数 ≈ 8400 × 4 - 296 = 33,304 个
正负样本比 ≈ 296 / 33,304 ≈ 0.89%
```

正样本占比不足 1%，其梯度信号在累积中被负样本淹没。

### 5.3 类别不平衡问题

71 个有标注的类别中，每个类别平均仅 13.3 个标注框，分布在约 10-15 张图片中。
这意味着：

- 每个类别在每个 batch 中出现的概率 ≈ 15/128 × 4 ≈ 47%
- 每次出现时，该类别的正样本数 ≈ 1-2 个 GT 框 × 10 = 10-20 个
- 相比 33,304 个负样本，单个类别的正样本梯度微乎其微

9 个完全缺失的类别（class 10, 12, 18, 19, 37, 47, 66, 70, 78）：
- 训练过程中永远没有正样本
- 对应的分类通道只接收负样本梯度
- logits 被持续推向更负的值
- 这些通道的 sigmoid 输出趋近于 0

### 5.4 与完整 COCO 数据集的对比

| 指标 | COCO128 | 完整 COCO train2017 |
|------|---------|---------------------|
| 训练图片数 | 128 | 118,287 |
| 总标注框数 | 941 | ~860,000 |
| 每个类别平均标注框数 | 13.3 | ~10,750 |
| 每个 batch 正样本数 | ~296 | ~296（相同 batch size） |
| 正负样本比 | 0.89% | 0.89%（单 batch 相同） |
| 每个类别每 batch 出现概率 | ~47% | ~100% |
| 每个类别累计正样本梯度 | 低（出现次数少） | 高（每 batch 都出现） |

关键差异不在于单个 batch 的正负样本比，而在于**训练过程中正样本梯度的累积量**。

完整 COCO 中，每个类别在几乎每个 batch 都有正样本，300 个 epoch × 每 epoch 约 29,500 个 batch = 约 885 万个 batch。每个类别累积的正样本梯度 = 885 万 × 每 batch 约 10~20 个正样本 × 单样本梯度。

COCO128 中，每个类别只有约 47% 的 batch 有正样本，累积梯度量约为完整 COCO 的 1/1000。

这个梯度量级的差异，决定了模型能否逃脱"低置信度死循环"。

## 6. 问题的深层本质

### 6.1 联合分配的耦合困境

TaskAlignedAssigner 将分类和回归的分配耦合在一起：

```python
alignment_metrics = (scores^α) * (ious^β) * mask_gt
```

这个设计的初衷是：让分类得分高且 IoU 大的预测点优先成为正样本，实现"分类与定位的对齐"（Task-Aligned）。

但在训练初期，这个设计创造了一个悖论：

- 要让分配器选出正确的正样本 → 需要模型先有合理的分类得分
- 要让模型学会合理的分类得分 → 需要分配器先选出正确的正样本

这就是"鸡生蛋、蛋生鸡"问题。在正常训练条件下（大数据集、预训练权重），这个循环能通过 IoU 分量的引导逐步打破。但在小数据集上，循环无法打破。

### 6.2 beta=6.0 的双刃剑效应

beta=6.0 意味着 IoU 的权重远高于分类得分。这是一个精心设计的超参数：

**优势**：在训练中后期，IoU 成为分配的主导因素，确保几何匹配准确的预测点成为正样本，即使其分类得分还不高。这让回归分支能先于分类分支收敛。

**劣势**：在训练初期，当分类得分趋近 0 时，alignment_metrics 主要由 ious^6.0 决定。即使 ious 只有 0.3，ious^6.0 = 0.000729，乘以 scores^0.5 ≈ 0 后，alignment_metrics 趋近 0。分配器选不出有意义的正样本，分类分支失去训练信号。

### 6.3 为什么这是架构级别的问题，而非超参数问题

即使调整以下超参数，问题也难以根本解决：

- **学习率**：增大 lr 会加速 logits 向负方向移动，加快进入死循环；减小 lr 会减慢学习速度，但无法改变正负样本梯度失衡的本质。
- **batch_size**：增大 batch_size 增加单 batch 正样本数，但同时也增加负样本数，正负比不变。
- **topk**：增大 topk 增加每个 GT 的正样本数，但对齐分数低的正样本贡献的梯度也很小。
- **alpha/beta**：减小 beta 可以降低 IoU 的权重，让 scores 的影响更大，但这会破坏"分类-定位对齐"的核心设计。

根本原因在于：**小数据集 + 联合分配 + 从零训练** 三者组合下的梯度失衡。这不是单一超参数能解决的。

## 8. 解决方案

### 方案 A：减少类别数（推荐）

将 nc 从 80 减少到 2~10，选择 COCO128 中标注最多的类别。
这直接增加了每个类别的平均标注数，提高了正样本梯度的累积量。

```
nc=2  → 每个类别约 470 个标注 → 正样本梯度增加约 35 倍
nc=10 → 每个类别约 94 个标注  → 正样本梯度增加约 7 倍
```

### 方案 B：使用预训练权重

加载 Ultralytics 官方 YOLOv8 预训练权重作为初始化，而非从零训练。
预训练模型的分类分支已有合理的权重分布，不会陷入"logits 全 0"的初始状态。

### 方案 C：添加分类损失权重预热

在训练初期增大分类损失的权重，强制模型优先学习分类：

```python
# 训练循环中
cls_weight = min(1.0, (epoch + 1) / 50)  # 前 50 个 epoch 线性增加
loss = loss_cls * cls_weight * 10.0 + loss_box * 1.5
```
