# 置信度阈值问题分析

## 事件描述

- **训练 50 epoch** + `conf_thres=0.1`：推理结果为"图片中没有检测到任何目标"
- **训练 100 epoch** + `conf_thres=0.0001`：成功检测出 1698 个目标

## 原因分析

### 1. 根本原因：模型训练不足，输出置信度过低

YOLOv8 的分类分支使用 `sigmoid` 激活函数，将 logits 压缩到 [0, 1] 区间。在训练初期，模型的权重尚未收敛，sigmoid 输出普遍接近 0.5（随机猜测水平）。

- **50 epoch 时**：模型对 COCO 80 类的学习不充分，真实目标的置信度可能只有 0.01~0.05，远低于 0.1 的阈值，全部被过滤掉。
- **100 epoch 时**：模型多训练了 50 轮，置信度有所提升，但仍然偏低，必须将阈值降到 0.0001 才能检出目标。

### 2. 次要原因：COCO128 数据集规模太小

COCO128 仅包含 128 张图片，却要学习 80 个类别的检测。每个类别平均只有 1~2 张训练样本，模型极难学到有效的类别特征。这导致：

- 模型对多数类别的分类置信度始终偏低
- 需要更多 epoch 才能达到可用的收敛水平
- 即使训练 100 epoch，效果仍然不理想

### 3. 置信度阈值过低的副作用

`conf_thres=0.0001` 虽然能检出目标，但检出了 1698 个框，远超一张图片中实际目标数量。这说明：

- 大量低质量的误检框被保留
- NMS 的 `iou_thres=0.45` 无法有效过滤不同位置的误检
- 模型的分类能力仍然很弱，无法区分目标和背景

## 解决方法

### 方案一：增加训练 epoch（推荐）

继续训练至 200~300 epoch，让模型充分收敛。随着训练推进，真实目标的置信度会逐渐提高，最终可以使用合理的阈值（如 0.25~0.5）。

```python
num_epochs = 300  # 增加到 300
```

### 方案二：使用更大数据集

将训练数据从 COCO128 扩展到完整的 COCO train2017（约 118k 张图片），模型能学到更丰富的特征，置信度会显著提升。

### 方案三：调整学习率策略

当前使用固定学习率 `lr=1e-3`，建议加入学习率调度器：

```python
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=num_epochs, eta_min=1e-6)
```

### 方案四：合理设置置信度阈值

在模型训练充分后，建议通过以下步骤确定最佳阈值：

1. 在验证集上计算不同阈值下的 Precision-Recall 曲线
2. 选择 F1-score 最高的阈值作为最终值
3. 通常 YOLO 系列模型使用 0.25~0.5 作为默认阈值

## 总结

| 状态 | Epoch | 阈值 | 结果 |
|------|-------|------|------|
| 训练不足 | 50 | 0.1 | 无检测 |
| 训练不足 | 100 | 0.0001 | 1698 个框（大量误检） |
| 训练充分（目标） | 300 | 0.25~0.5 | 合理检测数量 |

**核心结论**：问题的本质是训练不充分，而非阈值设置错误。降低阈值只是治标不治本，增加训练量才是根本解决方案。
